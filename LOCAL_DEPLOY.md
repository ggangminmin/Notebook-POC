# 로컬 서버 상태 그대로 배포하기

localhost:5174에서 테스트 중인 상태를 Git 커밋 없이 바로 배포하는 방법입니다.

## ✅ 방법 1: Vercel CLI로 직접 배포 (추천! 가장 빠름)

Git 커밋/푸시 없이 로컬 파일을 직접 배포합니다.

### 단계:

1. **프로젝트 폴더를 파일 탐색기에서 열기**
   - `notebooklm copy` 폴더에서 우클릭
   - "터미널에서 열기" 또는 "PowerShell에서 열기"

2. **배포 명령 실행**
   ```powershell
   # 프로덕션 배포 (이미 Vercel 프로젝트가 연결되어 있다면)
   vercel --prod
   
   # 또는 처음이라면
   vercel
   ```

**장점:**
- ✅ Git 커밋/푸시 불필요
- ✅ 즉시 배포 가능
- ✅ 로컬 파일 그대로 배포

---

## 방법 2: Git에 커밋 후 자동 배포

Vercel이 Git 저장소를 모니터링하고 있으면, 푸시하면 자동으로 배포됩니다.

### 단계:

1. **변경사항 확인**
   ```bash
   git status
   ```

2. **변경사항 추가 및 커밋**
   ```bash
   git add .
   git commit -m "최신 변경사항 배포"
   ```

3. **GitHub/GitLab에 푸시**
   ```bash
   git push
   ```

4. **Vercel 자동 배포 대기**
   - 푸시 후 몇 분 내 자동 배포 시작
   - Vercel 대시보드에서 진행 상황 확인

---

## 🤔 어떤 방법을 선택할까요?

- **지금 당장 빠르게 배포하고 싶다면** → **방법 1 (Vercel CLI)**
- **Git 히스토리를 관리하면서 배포하고 싶다면** → **방법 2 (Git 푸시)**
