# 🤖 AI 행동 지침 설정 - 사용자 가이드

## 📌 이 기능이 뭔가요?

**AI 행동 지침 설정**은 AI가 답변할 때 **어떤 역할과 스타일로 대답할지** 직접 지정할 수 있는 기능입니다.

### 🎭 쉬운 비유
- AI는 연극 배우와 같습니다
- 이 기능은 배우에게 **"어떤 캐릭터를 연기할지"** 알려주는 **대본(시나리오)**입니다
- 같은 질문이라도 **전문가**, **선생님**, **친구** 역할에 따라 답변이 달라집니다

---

## 🎯 왜 필요한가요?

### 문제 상황
일반 AI는 모든 질문에 비슷한 스타일로 답변합니다:
- 📄 **문서 분석**: "이 계약서의 핵심은 무엇인가요?"
- 💬 **AI 답변**: "이 문서는 계약에 관한 내용입니다. 주요 조항은..."

### 개선 후 (AI 지침 적용)
**"에이비딩 운영자"** 페르소나 설정 시:
- 📄 **문서 분석**: "이 입찰 공고의 핵심은?"
- 💬 **AI 답변**: "🏢 **입찰 담당자님께**,

  이 공고는 **전자입찰 방식**을 적용하며, 다음 3가지 필수 제출 서류를 **2025년 1월 15일 17:00까지** 제출해야 합니다:

  1. ✅ 사업자등록증 사본
  2. ✅ 재무제표 (최근 3년)
  3. ✅ 기술제안서

  ⚠️ **주의**: 기한 경과 시 자동 탈락 처리됩니다."

### 1️⃣ **우측 패널 찾기**
```
┌─────────────────────────────────────┐
│  왼쪽          중앙           우측   │
│  소스          채팅        ⬅ 여기!  │
│                                     │
│               스크롤 ⬇              │
│                                     │
│  ┌────────────────────────────┐    │
│  │ ⚙️ AI 행동 지침 설정       │    │
│  │ [에이비딩 운영자] [분석가] │    │
│  │ Textarea: ...              │    │
│  └────────────────────────────┘    │
└─────────────────────────────────────┘
```

### 2️⃣ **빠른 설정: 프리셋 버튼 클릭**
- **🏢 에이비딩 운영자**: 입찰/계약 관리 전문가 말투
  - 법적 용어 사용
  - 절차 단계별 설명
  - 기한/규정 강조

- **📄 일반 문서 분석가**: 중립적 분석가 말투
  - 객관적이고 간결한 요약
  - 사실만 제시
  - 해석 최소화

### 3️⃣ **적용하기**
1. 프리셋 버튼 클릭 (또는 직접 입력)
2. 하단 **[적용]** 버튼 클릭
3. ✅ "AI 지침이 적용되었습니다" 메시지 확인
4. 이제부터 AI가 설정한 스타일로 답변합니다!

## 💡 실전 활용 예시

### 예시 1: 입찰 공고 분석 (에이비딩 운영자 모드)

**설정 전 (일반 AI)**
```
Q: 이 입찰 공고의 주요 내용은?
A: 이 문서는 입찰 공고입니다. 주요 내용으로는 입찰 기간, 제출 서류 등이 있습니다.
```

**설정 후 (에이비딩 운영자 페르소나)**
```
Q: 이 입찰 공고의 주요 내용은?
A: 🏢 **입찰 담당자님께**,

이 공고의 **필수 확인 사항**은 다음과 같습니다:

### 📅 일정
- 공고일: 2025.01.05
- 제출 마감: 2025.01.15 17:00 (평일 기준)
- 개찰일: 2025.01.20 14:00

### 📋 제출 서류 (필수)
1. 사업자등록증 원본 또는 사본
2. 법인인감증명서 (발급 3개월 이내)
3. 기술제안서 (A4 20매 이내)

### ⚠️ 주의사항
- 기한 엄수 (1분 초과 시 무효 처리)
- 전자입찰 시스템 사전 등록 필수
- 문의: procurement@example.com

📌 추가 질문이 있으시면 말씀해주세요!
```

### 예시 2: 기술 문서 분석 (일반 분석가 모드)

**설정 전**
```
Q: 이 기술 문서의 핵심은?
A: 이 문서는 API 사용법을 설명합니다. 여러 기능이 있네요!
```

**설정 후 (일반 문서 분석가)**
```
Q: 이 기술 문서의 핵심은?
A: ### 문서 요약

**문서 유형**: REST API 기술 명세서
**버전**: v2.1.3
**페이지 수**: 45p

### 핵심 내용
1. **인증 방식**: OAuth 2.0 Bearer Token
2. **주요 엔드포인트**:
   - GET /api/users (사용자 조회)
   - POST /api/orders (주문 생성)
3. **응답 형식**: JSON
4. **요청 제한**: 시간당 1,000건

### 기술 요구사항
- HTTPS 필수
- Content-Type: application/json
- 최소 Node.js 18.0 이상

📄 출처: 3-5페이지, 12페이지, 23-25페이지
```

## 🎨 커스텀 지침 만들기

### 직접 입력하는 방법

프리셋이 아닌 **나만의 AI 캐릭터**를 만들 수 있습니다!

#### 📝 입력 예시
```
당신은 친절한 법률 자문 AI입니다.

**역할**:
- 법률 용어를 쉽게 풀어서 설명
- 일반인도 이해할 수 있는 예시 사용
- 위험 요소는 반드시 강조

**응답 스타일**:
- 존댓말 사용
- 중요한 부분은 굵게 표시
- 법률 조항 인용 시 출처 명시

**주의사항**:
- 확실하지 않은 법률 해석은 "전문가 상담 권장" 명시
```

#### 💬 적용 후 대화 예시
```
Q: 계약서의 손해배상 조항이 이해가 안 돼요.
A: 안녕하세요! 해당 조항을 쉽게 설명드리겠습니다.

**손해배상 조항 (제12조)** 의미:

"계약 위반 시 상대방에게 **실제 발생한 손해**를 보상해야 합니다."

### 💡 쉬운 예시
만약 A회사가 납기를 어겨서 B회사가 100만원 손해를 봤다면,
→ A회사는 B회사에게 100만원을 배상해야 합니다.

### ⚠️ 주의!
본 조항은 **"위약금"과 별개**입니다. 위약금은 제15조를 참고하세요.

📌 복잡한 사안이므로 **변호사 상담**을 권장드립니다.
```

## 🔧 활용 팁

### ✅ 이렇게 사용하세요

1. **업무별로 프리셋 전환**
   - 입찰 업무 → "에이비딩 운영자"
   - 계약서 검토 → "법률 자문" (커스텀)
   - 기술 문서 → "일반 분석가"

2. **명확한 역할 정의**
   ```
   좋은 예:
   "당신은 20년 경력의 계약 관리 전문가입니다.
    위험 요소를 먼저 지적하고, 대응 방안을 제시합니다."

   나쁜 예:
   "친절하게 대답해주세요." (너무 모호함)
   ```

3. **원하는 형식 지정**
   ```
   "모든 답변은 다음 구조로 작성:
   1. 핵심 요약 (2줄)
   2. 상세 분석 (3-5개 항목)
   3. 행동 조치 (체크리스트)"
   ```

### ❌ 피해야 할 실수

1. **너무 긴 지침** (AI가 혼란스러워함)
   - ❌ 10줄 이상의 복잡한 규칙
   - ✅ 3-5줄로 핵심만 정리

2. **모순된 요구**
   - ❌ "간결하게 + 모든 세부사항 포함"
   - ✅ "핵심만 3줄 요약 + 상세 항목은 리스트로"

3. **지침 변경 후 미적용**
   - 텍스트만 바꾸고 **[적용] 버튼 안 누름**
   - → AI가 이전 설정으로 계속 답변

## 🧪 테스트 시나리오

### 실습: 단계별 따라하기

#### Step 1: 프리셋 테스트
1. 우측 패널 → "AI 행동 지침 설정"
2. **[에이비딩 운영자]** 클릭
3. **[적용]** 클릭
4. 채팅에 질문: "입찰 참가 자격은?"
5. AI 답변 확인 → 전문 용어 + 절차 중심

#### Step 2: 다른 프리셋으로 전환
1. **[일반 문서 분석가]** 클릭
2. **[적용]** 클릭
3. 같은 질문 반복: "입찰 참가 자격은?"
4. 답변 비교 → 객관적이고 간결한 스타일

#### Step 3: 커스텀 지침 만들기
1. Textarea에 입력:
   ```
   당신은 신입 사원을 교육하는 선배입니다.
   어려운 용어는 쉽게 풀어서 설명하고,
   예시를 많이 들어주세요.
   ```
2. **[적용]** 클릭
3. 질문: "계약서 검토는 어떻게 하나요?"
4. 답변 확인 → 교육적이고 친절한 톤

## ❓ 자주 묻는 질문 (FAQ)

### Q1. 지침을 여러 개 저장할 수 있나요?
**A**: 현재는 1개만 활성화됩니다. 자주 쓰는 지침은 메모장에 복사해두고 붙여넣기로 전환하세요.

### Q2. 적용하면 언제까지 유지되나요?
**A**: 브라우저를 새로고침하면 초기화됩니다. (향후 업데이트 예정)

### Q3. AI가 지침을 무시하는 것 같아요.
**A**: 다음을 확인하세요:
- **[적용]** 버튼을 눌렀는지
- 지침이 너무 모호하지 않은지
- 문서 내용과 지침이 충돌하지 않는지

### Q4. 영어로 지침을 써도 되나요?
**A**: 네! AI는 한글/영어 모두 이해합니다.

### Q5. 지침이 너무 길면 어떻게 되나요?
**A**: AI가 핵심만 따르거나 일부를 놓칠 수 있습니다. **5줄 이내** 권장.
